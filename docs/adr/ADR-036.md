# S3 Only as Year 1 File Storage Strategy

## Status
Accepted

## Date
2026-07-14

## Context
RentSecure requires file storage for property images, rent agreements, receipts, and user uploads. CloudFront CDN improves user experience by caching files at edge locations, but it adds $1–2/month in costs and configuration complexity. For a solo founder with a single-region deployment (ap-south-1) targeting Indian users, S3 direct access provides acceptable performance.

## Decision
Year 1 uses Amazon S3 only for file storage. CloudFront CDN is introduced in Stage 2 when users report slow image loading or bandwidth costs exceed 10 GB/month. Private files are accessed via Django proxied views or presigned S3 URLs.

## Consequences
- Zero additional CDN cost
- Simple S3 configuration and IAM policies
- Direct S3 URLs for public files
- Presigned URLs for private files
- No edge caching benefits
- Triggers Stage 2 migration when:
  - Users report slow image loading
  - Bandwidth costs exceed 10 GB/month
  - Global users require edge acceleration
